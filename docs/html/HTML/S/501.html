  52 /* A client will create one epan_t for an entire dissection session.
  53  * A single epan_t will be used to analyze the entire sequence of packets,
  54  * sequentially, in a single session. A session corresponds to a single
  55  * packet trace file. The reaons epan_t exists is that some packets in
  56  * some protocols cannot be decoded without knowledge of previous packets.
  57  * This inter-packet "state" is stored in the epan_t.
  58  */
