   4 /** \mainpage oflops 
   5  *
   6  * oflops (OpenFlow operations per second) is created to 
   7  * benchmark an OpenFlow switch.
   8  *
   9  * @author Rob Sherwood
  10  * @date September 2009
  11  *
  12  *
  13  * Read the README file for instructions on building, setting up, and running oflops.
  14  *
  15  * \section test_module Writing New Modules
  16  * Test modules are event driven.  To write a new test module, you need to create a shared library that defines the appropriate callbacks.
  17  * Most callbacks have reasonable defaults, so new test modules do not have to define all callbacks.
  18  *
  19  * The list of callbacks are documented in <a href="structtest__module.html">struct test_module</a> and 
  20  * the API exposed to the test module is available in test_module.h .
  21  *
  22  * <i>Channels</i> are a critical concept in oflops.  Channels a roughly equivalent to interfaces, and there are
  23  * both control and data channels.  
  24  *
  25  * \subsection control_channel Control Channel
  26  *  The control channel is a reference to the openflow tcp control connection.  Each oflops run should only have one control channel defined.
  27  *  Openflow message from the switch are sent to oflops via one of the of_event_* callbacks and control messages are sent to the switch
  28  *  via the <a href="test__module_8h.html#84e360ed0ef80457bfa72b64e6b3dbea">oflops_send_mesg()</a> function.
  29  * \subsection data_channel Data Channel(s)
  30  *  A data channel is a reference to a raw interface connected (directly or logically) to the switch.  
  31  *  Each oflops run can define zero or more data channels.  Data channels are used for sourcing and syncing 
  32  *  emulated user traffic. Oflops is notified of new packets on the data channel via the <a href="test__module_8h.html#84e360ed0ef80457bfa72b64e6b3dbea">handle_pcap_event()</a> callback, and typically send outgoing data messages <a href="test__module_8h.html#84e360ed0ef80457bfa72b64e6b3dbea">oflops_send_raw_mesg()</a> function.
  33  *
  34  * \section example_modules Example modules
  35  * <UL>
  36  * <LI> debug.c is used for debugging oflops </LI>
  37  * <LI> pktin.c is test the number of packet in that can be
  38  *      provided by a switch and the associated delay. </LI>
  39  * </UL>
  40  */
